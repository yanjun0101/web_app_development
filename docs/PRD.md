# 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
現代人常在不同平台（部落格、YouTube、Instagram 各大社群）看到喜歡的食譜，但缺乏一個統一整合的地方來儲存與分類，導致要下廚時經常找不到之前存下的食譜。因此，打造一個統整各處食譜的個人專屬收藏夾，能幫助使用者更有系統地管理料理靈感。

### 目標用戶
- 喜歡在家做料理、經常參考網路食譜的自煮族
- 需要每日準備三餐的家庭主婦/主夫
- 想嘗試新菜色的料理愛好者

### 核心價值主張
集中管理散落各地的食譜連結與紀錄，提供標籤分類與快速搜尋功能，讓下廚不再為了「找食譜」或「不知道要吃什麼」而煩惱。

---

## 2. 功能需求

以下列出本系統的主要功能，並透過使用者故事 (User Story) 描述：

1. **新增/收藏食譜**
   - 作為 **料理愛好者**，我希望 **能手動建立食譜資訊（名稱、連結、備註）**，以便 **我將有興趣的料理記錄下來**。
2. **食譜列表與詳細資訊瀏覽**
   - 作為 **料理愛好者**，我希望 **能看到所有已收藏食譜的清單與其詳細內容**，以便 **我挑選和參閱想做的料理**。
3. **標籤分類與篩選**
   - 作為 **料理愛好者**，我希望 **能為食譜加上自訂標籤（如：台式、甜點、氣炸鍋、20分鐘內完成）並且可以透過這些標籤進行篩選**，以便 **我依據當下的情境或現有食材快速縮小選擇範圍**。
4. **食譜關鍵字搜尋**
   - 作為 **料理愛好者**，我希望 **能輸入關鍵字來搜尋食譜名稱或備註**，以便 **我能快速定位到特定的某一道菜**。
5. **隨機推薦「今晚吃什麼」**
   - 作為 **不知道要吃什麼的自煮族**，我希望 **系統能從我的收藏中隨機推薦一道料理**，以便 **我能突破選擇困難，節省決定菜單的時間**。
6. **編輯與刪除食譜**
   - 作為 **料理愛好者**，我希望 **可以修改現有食譜的內容或將不再感興趣的食譜移除**，以便 **我維持收藏夾的最新狀態和整潔**。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask (Python)
- **前端模板**：Jinja2 + 基本的 HTML/CSS/JS (可選配 Bootstrap 等輕量框架增進美觀)
- **資料庫**：SQLite (適合本地或小型專案使用)

### 效能與安全考量
- **效能**：食譜列表應能快速載入，未來若資料筆數增多，列表需考慮分頁。
- **安全**：防範常見的 Web 程式漏洞，如：SQL Injection (透過 ORM 或 Parameterized Query 處理)、XSS (Jinja2 預設會過濾 HTML 跳脫字元)。

## 4. MVP 範圍 (Minimum Viable Product)

| 優先級 | 範圍層級 (MoSCoW) | 功能項目 |
| :---: | :--- | :--- |
| 高 | **Must Have** (必須有) | - 手動新增/編輯/刪除食譜<br>- 以列表檢視所有食譜<br>- 查看單一食譜的詳細內容 |
| 中 | **Should Have** (應該有) | - 標籤系統（為食譜增加標籤並能點擊篩選）<br>- 食譜關鍵字搜尋 |
| 低 | **Nice to Have** (最好有) | - 隨機抽取一道食譜推薦給使用者<br>- 貼上網址自動抓取標題/圖片功能<br>- 支援圖片上傳作為食譜封面 |
